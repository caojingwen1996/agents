# 个人规则沉淀提示词

使用本提示词处理用户关于规则总结、沉淀、升级、废弃和维护的请求。

## 读取并遵守

1. `agent.md`
2. `prompts/system.md`
3. `templates/candidate_rule.md`

## 输入

用户可能提出：

1. 这次复盘能沉淀什么规则？
2. 帮我总结一条规则。
3. 这条规则适合我吗？
4. 把这个加入候选规则。
5. 这个规则以后要不要保留？
6. 把候选规则升级为当前个人规则。
7. 这条规则是不是应该废弃？

不要把外部参考规则或单次交易经验直接升级为当前个人规则。

## 执行流程

1. 识别用户想总结、沉淀、升级、废弃还是维护规则。
2. 回顾相关交易前审计。
3. 回顾相关交易后复盘。
4. 判断规则来源：外部参考规则、候选个人规则、当前个人规则、废弃规则。
5. 如规则来源涉及 llmwiki，调用 `obsidian_kb` skill 查询来源。
6. 判断该规则是否经过多次验证。
7. 判断该规则是否匹配用户资金、时间、性格和交易周期。
8. 判断该规则是否有明确适用场景和不适用场景。
9. 判断该规则是否能降低冲动交易、重大亏损或系统混用风险。
10. 输出候选个人规则记录。
11. 如果用户要求升级，必须再次要求用户明确确认。

## 规则来源判断

必须区分：

1. 外部参考规则：来自 llmwiki，只能作为参考。
2. 候选个人规则：来自审计、复盘或行为统计，尚未正式生效。
3. 当前个人规则：用户已明确确认采用，具有最高优先级。
4. 废弃规则：用户已确认废弃，或经复盘证明不适合当前系统。

## 升级条件

候选个人规则升级为当前个人规则前，至少检查：

1. 是否经过多次交易前审计使用。
2. 是否经过交易后复盘验证。
3. 是否由用户明确确认采用。
4. 表述是否清晰。
5. 适用场景是否明确。
6. 不适用场景是否明确。
7. 执行条件是否明确。
8. 违反后的处理方式是否明确。

不满足以上条件时，只能保留为候选个人规则或待观察规则。

## 优先追问信息

每轮最多追问 5 个问题。优先追问：

1. 这条规则来自哪次审计或复盘？
2. 它是否已经在多次交易中验证过？
3. 它适用于哪些交易系统、标的和周期？
4. 哪些情况下不适用？
5. 你是否确认要把它升级为当前个人规则？

## 输出状态

规则沉淀只能输出以下状态之一：

1. 外部参考规则。
2. 候选个人规则。
3. 当前个人规则。
4. 废弃规则。

未经用户明确确认，不得输出“当前个人规则”。

## 输出要求

候选规则记录使用 `templates/candidate_rule.md` 的结构。

输出中必须包含：

1. 规则摘要。
2. 规则来源。
3. 触发条件。
4. 执行动作。
5. 验证记录。
6. 升级建议。

如果规则来源涉及 llmwiki 但无法调用 `obsidian_kb`，必须标记“未检索 llmwiki，规则来源待补充”。

## 禁止事项

不得把 llmwiki 外部参考规则自动升级为个人规则。

不得把单次盈利经验沉淀为强规则。

不得把单次亏损经验直接废弃为无效规则。

不得在缺少验证记录时输出当前个人规则。

不得替用户确认规则升级。
