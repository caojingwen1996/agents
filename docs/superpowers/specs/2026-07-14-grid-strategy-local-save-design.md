# 网格策略本地保存设计

## 目标

为现有离线网格策略生成器增加策略保存能力。用户手工保存已经成功生成的方案后，工具下载一份与现有“导出完整方案 CSV”相同格式的文件，并在页面左侧保留可再次载入的策略索引。

本功能继续保持单页、离线和无服务器运行方式，不改变网格计算、留利润计算或现有 CSV 导出的口径。

## 用户流程

1. 用户填写参数并成功生成网格计划。
2. 用户点击“保存策略”。
3. 工具校验标的名称和最近一次成功生成的方案。
4. 工具下载一份现有完整方案格式的 CSV。
5. 工具把该方案的输入参数和保存时间写入浏览器本地索引。
6. 左侧列表立即显示或更新该标的。
7. 用户点击左侧策略时，工具恢复全部输入参数并立即重新生成结果。
8. 用户可以删除左侧记录；删除前必须确认。

## 页面结构

### 桌面端

页面使用左右双栏：

- 左侧为“已保存策略”列表。
- 右侧保留现有网格策略参数、压力测试、完成一轮汇总和网格表格。

左侧区域沿用当前页面的白色卡片、圆角、边框、字体和蓝色交互色，不引入新的视觉体系。列表只承担已保存策略的导航功能，不放置生成或编辑表单。

### 窄屏

在窄屏下改为上下排列，已保存策略位于参数区上方。页面不得出现由双栏布局造成的横向滚动，现有结果表格仍按当前方式处理自身宽度。

### 保存操作

“生成网格计划”附近增加“保存策略”按钮。生成仍是主操作，保存使用次级按钮样式。没有最近一次成功生成的方案时，保存按钮不可用；参数在生成后被修改时，不得把旧结果和新参数混合保存。

保存成功或失败使用现有错误提示区域附近的简短状态提示。提示使用 `aria-live="polite"`，不主动抢走键盘焦点。

## 左侧策略列表

每条记录显示：

- 标的名称；
- 最近保存时间；
- 删除按钮。

列表按最近保存时间倒序排列。点击记录主体会载入该策略；删除按钮与载入区域分离，避免误触。当前载入的记录使用边框、背景和文字权重共同表示选中状态，不能只依赖颜色。

没有保存记录时显示明确空状态：“暂无已保存策略，生成方案后可保存。”

删除按钮必须具有包含标的名称的可访问名称。所有交互元素支持键盘操作并保留清晰焦点轮廓，点击区域不小于当前页面的标准按钮高度。

## 保存和覆盖规则

### 唯一标识

保存前去除标的名称首尾空白。整理后的完整标的名称是唯一标识。名称完全相同时覆盖原索引记录，不保留多个历史版本；名称不同则新增记录。

覆盖不再二次确认。保存时间和全部输入参数更新，列表仍只保留一条该标的记录。

### 保存内容

本地索引保存重新生成方案所需的全部输入：

- 标的名称；
- 起始价；
- 步长比例；
- 最大跌幅；
- 资金模式；
- 总资金或每格金额；
- 单边费率；
- 留利润策略；
- 最近保存时间；
- 数据版本号。

计算结果不作为权威数据保存。载入时使用保存参数重新调用现有计算逻辑，避免计算口径升级后继续展示旧结果。

### 浏览器索引

索引使用 `localStorage`，以一个带版本号的 JSON 结构保存。读取时必须校验顶层结构和每条记录；无法识别或损坏的记录应跳过，并向用户提示本地记录无法完整读取，而不是使整个页面失效。

该索引仅用于生成左侧列表，不替代下载的 CSV 文件。清除当前浏览器的网站数据会删除索引。

### CSV 文件

保存策略复用现有 CSV 序列化和下载逻辑，确保内容、UTF-8 BOM、列结构和现有“导出完整方案 CSV”完全一致。

文件名继续使用：

```text
标的名称-网格策略-日期时间.csv
```

浏览器下载的每个 CSV 是独立文件。同名策略覆盖本地索引时仍会下载新文件；删除左侧记录不会删除电脑上已经下载的 CSV。

## 状态一致性

最近一次成功生成的方案继续作为导出和保存的唯一数据源。

- 初始进入页面时，没有可保存方案。
- 生成成功后，保存按钮可用。
- 任一影响计算的输入被修改后，当前结果视为过期，保存按钮重新不可用，直到再次生成成功。
- 载入左侧策略时，先恢复全部参数，再自动生成；生成成功后该方案可以再次保存。
- 生成失败时不得覆盖最近的本地策略索引，也不得下载 CSV。

## 删除行为

点击删除后使用浏览器确认对话框说明即将删除的标的名称。确认后只删除浏览器索引中的对应记录；取消则不做任何修改。

如果被删除的记录正处于选中状态，只取消左侧选中标记，不清空右侧已生成的结果，避免破坏用户当前查看内容。

## 异常处理

- 标的名称为空：不保存，提示先填写标的名称并重新生成方案。
- 没有成功生成的方案：不保存，提示先生成网格计划。
- 参数修改后结果已过期：不保存，提示重新生成。
- `localStorage` 写入失败：提示左侧索引未保存；CSV 下载是否执行成功单独反馈。
- CSV 生成或下载失败：提示文件未保存；不得把失败描述为成功。
- 本地索引 JSON 损坏：保留可识别记录，跳过损坏记录并提示。

保存是两个可独立失败的动作：下载 CSV 和更新浏览器索引。界面反馈必须分别反映真实结果，不使用单一成功提示掩盖部分失败。

## 不在本次范围

- 不增加 CSV 或 Excel 导入。
- 不扫描或读取浏览器下载目录。
- 不保存多个同名历史版本。
- 不增加策略搜索、排序切换、标签、云同步或跨浏览器同步。
- 不改为真正的 `.xlsx` 工作簿；继续使用当前可由 Excel 打开的 CSV 格式。
- 不删除用户电脑中已经下载的策略文件。

## 验证标准

- 保存成功后，左侧立即出现相应标的，并下载与现有导出内容一致的 CSV。
- 同名再次保存后，列表仍只有一条，参数和保存时间已更新。
- 不同标的分别保存为不同记录。
- 点击记录后恢复所有输入并自动生成与保存参数一致的结果。
- 删除确认后记录消失，取消删除时记录保持不变。
- 刷新网页后，策略列表仍存在。
- 空名称、未生成和参数已修改三种状态均不能保存，并提供明确提示。
- 损坏的本地索引不会阻止页面启动或正常生成新策略。
- 桌面端为左右双栏，窄屏为上下排列且无额外横向滚动。
- 保存、载入、删除均可用键盘完成，焦点状态和选中状态清晰。
- 现有网格 1.0、留利润、压力测试和 CSV 导出测试继续通过。
