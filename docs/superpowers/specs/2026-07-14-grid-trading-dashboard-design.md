# 网格交易收益仪表盘设计

## 1. 目标与范围

在当前仓库中新增一个独立的本地网页工具。工具固定读取 `data/交易记录.xlsx`，自动获取单只 A 股的历史日线行情，并生成与参考图一致的黑色收益仪表盘。用户只需维护 Excel，双击启动工具或在页面点击“刷新数据”即可查看最新结果。

首版只支持：

- 单只 A 股；
- 一份固定位置的 Excel；
- 买入、卖出和现金分红；
- 同一网格编号内的买卖配对；
- 本地 Windows 运行；
- AKShare 历史行情与本地缓存。

首版不包含多股票组合、在线部署、券商自动同步、用户账户系统或交易下单功能。

## 2. 技术方案

采用 Flask + ECharts：

- Flask 提供本地 HTTP 服务、刷新接口和页面渲染；
- pandas 与 openpyxl 读取、校验 Excel；
- AKShare 获取 A 股历史日线行情；
- Python 计算持仓、现金流、XIRR、回撤与网格收益；
- ECharts 绘制价格线、成本线、买卖点和提示框；
- CSS 复刻参考图的黑色背景、白色边框、主图和右侧指标卡片；
- `启动工具.bat` 启动服务并自动打开默认浏览器。

新工具使用独立目录，不修改仓库内现有的回测、技能或插件代码。交易文件和行情缓存均保存在本机。

## 3. 目录与运行方式

工具目录包含以下职责明确的部分：

- 应用入口：启动 Flask、打开浏览器；
- Excel 读取与校验：把工作簿转换成标准交易记录；
- 行情服务：按股票代码、日期范围和复权方式读取或补充缓存；
- 计算模块：持仓成本、现金流、网格配对和指标；
- 页面模板与静态资源：仪表盘布局、样式和 ECharts 配置；
- `data/交易记录.xlsx`：用户维护的固定输入文件；
- `data/cache/`：行情缓存；
- `启动工具.bat`：Windows 双击入口。

启动流程：

1. 双击 `启动工具.bat`；
2. 本地服务读取并校验 Excel；
3. 行情服务读取缓存并补充缺少的交易日；
4. 计算模块生成图表序列和指标；
5. 浏览器自动打开仪表盘。

修改 Excel 后，用户点击页面右上角的“刷新数据”。刷新操作重新读取 Excel、补充行情并重新计算，无需重启服务。

## 4. Excel 输入契约

固定文件为 `data/交易记录.xlsx`，包含 `配置` 和 `交易流水` 两个工作表。

### 4.1 配置

| 字段 | 必填 | 示例 | 规则 |
| --- | --- | --- | --- |
| 股票代码 | 是 | 600519 | 六位 A 股代码，按文本读取并保留前导零 |
| 图表标题 | 是 | 贵州茅台提款足迹 | 显示在主图顶部 |
| 行情复权 | 是 | 不复权 | 首版固定只接受“不复权”，保证行情价格与实际成交价口径一致 |
| 策略预算 | 是 | 500000 | 正数，用于计算单品仓位 |

股票名称由行情数据自动获取，不要求用户填写。建仓日期由首笔买入记录推导，XIRR 计算日由最新有效行情日确定。

### 4.2 交易流水

| 列名 | 必填范围 | 示例 | 规则 |
| --- | --- | --- | --- |
| 日期 | 全部 | 2025-01-10 | 合法日期；同日记录保留 Excel 原顺序 |
| 类型 | 全部 | 买入 | 只允许买入、卖出、分红 |
| 网格编号 | 买入、卖出 | G3 | 同一格买卖使用相同编号；分红可为空 |
| 数量 | 买入、卖出 | 100 | 正数；分红填 0 或留空 |
| 成交价 | 买入、卖出 | 15.00 | 正数；分红填 0 或留空 |
| 手续费 | 全部 | 5.00 | 非负数，没有则填 0 |
| 印花税 | 全部 | 1.60 | 非负数，没有则填 0 |
| 现金金额 | 分红 | 300.00 | 分红为正数；买卖填 0 或留空 |
| 备注 | 否 | 触发第三格 | 自由文本，不参与计算 |

工具附带格式正确的模板和示例记录。数据按日期升序处理；同一天按 Excel 行号处理，确保计算顺序可复现。

## 5. 交易与网格计算

### 5.1 现金流

- 买入现金流：`-(数量 × 成交价 + 手续费 + 印花税)`；
- 卖出现金流：`数量 × 成交价 - 手续费 - 印花税`；
- 分红现金流：`现金金额 - 手续费 - 印花税`；
- 期末现金流：在最新行情日加入当前剩余持仓市值，作为 XIRR 的终值；
- 若最新行情日与已有现金流同日，将同日现金流合并后计算。

### 5.2 整体持仓成本

整体剩余持仓使用移动加权平均法：

- 买入时，将买入金额、手续费和印花税加入剩余持仓成本；
- 卖出时，按卖出前的单位移动平均成本转出对应数量的成本；
- 卖出费用不改变剩余持仓成本，计入已实现收益；
- 分红不改变持仓数量和剩余持仓成本。

任何一步都不允许整体持仓数量为负。

### 5.3 网格配对

每个网格编号维护独立的买入批次队列。卖出在同一网格内按先进先出配对：

- 支持一次买入对应多次卖出；
- 支持多次买入对应一次卖出；
- 同一网格可以完成多轮交易；
- 一笔卖出完全关闭该网格当时的全部剩余数量时，记为一次“完成”；
- 卖出数量超过该网格已买入未卖出数量时，整次计算失败并报告对应行；
- 买卖费用按该行实际值计入；分批配对时，费用按配对数量比例分摊。

网格已实现净收益等于匹配卖出的净收入，减去所匹配买入批次的含费成本。单格收益率等于该网格累计已实现净收益除以已匹配买入含费成本。平均持有天数按匹配数量加权。

## 6. 图表设计

页面主视图区复刻参考图：黑色背景、白色外框、左侧主图、右侧纵向指标卡片、底部图例。桌面浏览器宽度不足时允许整体按比例缩小，保持左右关系，不改成移动端卡片流。

主图从首笔买入日开始：

- 灰色价格线：`每日收盘价 ÷ 首笔买入成交价 - 1`；
- 红色虚线成本线：`当日移动平均单位成本 ÷ 首笔买入成交价 - 1`；
- 红色圆点：买入，标签为 `买 G3`；
- 绿色圆点：卖出，标签为 `卖 G3`；
- 横轴为交易日，纵轴为百分比；
- 悬停提示显示日期、收盘价、价格涨跌幅、单位成本，以及当天的交易信息。

没有持仓的日期不绘制成本线。交易日期不是交易日时，交易标记落在该日期之后第一个有行情的交易日；提示框仍显示实际成交日期和成交价。

主图下方增加网格统计表，包含网格编号、完成次数、已卖出数量、已实现净收益、单格收益率和平均持有天数。该表位于参考图式主视图区下方，不改变首屏仪表盘的布局。

## 7. 指标口径

右侧指标卡片依次显示：

1. 持仓收益率：`(当前市值 - 剩余持仓成本) ÷ 剩余持仓成本`；空仓时显示 `—`；
2. XIRR 年化：使用全部交易、分红和期末持仓市值现金流按实际日期计算；无法形成至少一正一负现金流时显示 `—`；
3. 现距建仓涨跌：`最新收盘价 ÷ 首笔买入成交价 - 1`；
4. 距建仓最大跌幅：`建仓后最低收盘价 ÷ 首笔买入成交价 - 1`；
5. 提款次数：存在有效匹配数量的卖出记录笔数；
6. 单品仓位：`当前持仓数量 × 最新收盘价 ÷ 策略预算`；
7. 运行天数：最新行情日与首笔买入日的自然日差；
8. 建仓日期：首笔买入日期；
9. XIRR 计算日：最新有效行情日期。

网格统计表中的“完成次数”与“提款次数”口径不同：提款次数按有效卖出记录计数，完成次数按某个网格被卖至零持仓的完整轮次计数。

## 8. 行情获取与缓存

行情服务使用 AKShare 获取目标股票从首笔买入日前一个交易日至当前日期的不复权日线。缓存键由股票代码、周期和复权口径组成。

- 首次运行拉取完整区间并写入 `data/cache/`；
- 后续刷新只请求缓存最后日期之后的数据，再去重合并；
- 请求失败但缓存覆盖全部交易日期时，使用缓存生成报告，并在页面显示“行情截至 YYYY-MM-DD”；
- 请求失败且缓存不能覆盖全部交易日期时，不生成可能误导的最新指标，保留最近一次成功页面并显示错误；
- 原始交易流水不写入缓存，也不发送到行情源。

## 9. 校验与错误展示

Excel 校验失败时，错误消息必须包含工作表、Excel 行号、字段名和原因。以下情况阻止重新计算：

- 文件或必需工作表不存在；
- 配置字段缺失或值非法；
- 交易列缺失；
- 日期、数量、成交价或金额格式非法；
- 买卖记录缺少网格编号；
- 未出现任何买入记录；
- 整体持仓或单个网格出现超额卖出；
- 行情缺失且缓存不足；
- XIRR 以外的核心指标无法形成。

刷新失败时，页面顶部显示错误，但保留最近一次成功结果。工具不猜测、不跳过、不自动修正错误交易。
