# 网格策略 1.0 计划生成器设计

## 背景

本工具依据《波段策略·网格之一》和《波段策略·网格之二：网格策略基础/1.0 版》设计。文章将网格定位为中短期波段策略，用于获取波动利润和缓解长期持仓焦虑；它只能是投资体系中的配角，不能替代长期策略。首版严格实现文章 1.0 的计划生成规则，不自行引入 2.0/3.0 的趋势、底仓或动态网格逻辑。

## 目标

提供一个无需安装、可直接在本地打开的网页。用户手工输入标的与资金参数后，工具生成：

- 固定步长的买入价和对应卖出价；
- 按 100 份取整的每档买卖数量；
- 税费前、税费后的单格利润；
- 覆盖最大跌幅所需的资金与最坏情形压力测试。

## 非目标

首版不包含：

- 实时行情、自动估值或标的推荐；
- 实际成交记录、当前持仓维护或交易提醒；
- CSV、PDF 或其他导出；
- 强制风险确认清单；
- 最小交易单位配置，统一使用 100 份；
- 网格 2.0/3.0、复利百分比网格、动态步长或回测。

## 技术形态

工具放在独立目录 `grid-strategy-generator/`，核心交付物为可双击打开的 `index.html`。页面不请求网络，所有计算都在浏览器本地完成，不保存用户输入。

为保持单文件使用体验，计算函数内嵌在 HTML 中。测试脚本从 HTML 的明确标记区间加载纯计算函数，并使用 Node 内置测试能力验证，不增加第三方依赖。

## 页面结构

采用已确认的线性布局 A：

1. 标题与简短规则说明；
2. 参数输入区；
3. “生成网格计划”按钮；
4. 压力测试摘要；
5. 完整网格表。

初次打开时只显示说明和参数区。生成后修改参数不会自动改变旧结果；用户必须再次点击生成按钮，避免把旧表误认为新计划。

## 输入参数

| 参数 | 说明 | 默认值 |
|---|---|---:|
| 标的名称 | 仅作计划标题，不参与计算 | 空 |
| 起始价 | 第一档买入价，也是步长计算基准 | 必填 |
| 步长比例 | 价格步长占起始价的比例 | 5% |
| 最大跌幅 | 压力测试至少需要覆盖的下跌幅度 | 必填 |
| 资金模式 | 固定每格金额 / 限定总资金 | 限定总资金 |
| 每格金额 | 固定每格金额模式下使用 | 按模式显示 |
| 总资金 | 限定总资金模式下使用 | 按模式显示 |
| 单边费率 | 买入和卖出分别收取 | 0% |

页面提供 5%“普通”和 10%“高波动”快捷步长，但最终仍以输入框数值为准。

## 网格计算规则

### 1. 固定步长

文章 1.0 使用等差价格网格，不使用逐档复利跌幅。

```text
原始价格步长 = 起始价 × 步长比例
网格档数 = ceil(最大跌幅 ÷ 步长比例) + 1
第 i 档原始买入价 = 起始价 - i × 原始价格步长，i 从 0 开始
```

网格档数包含起始档。若最大跌幅不是步长的整数倍，向下多生成一档，以保证实际覆盖不小于目标。例如最大跌幅 43%、步长 5%，理论覆盖 45%。

### 2. 报价精度

所有买卖价格四舍五入到小数点后三位：

```text
第 i 档买入价 = round(第 i 档原始买入价, 3)
第 0 档卖出价 = round(起始价 + 原始价格步长, 3)
第 i 档卖出价 = 第 i-1 档已取整买入价，i > 0
```

这样每一档的卖出价始终与表格中上一档买入价完全一致。

### 3. 资金模式

两种资金模式互斥：

- 固定每格金额：用户输入的是每格用于买入证券的本金预算；工具另行计算买入费用和实际所需现金。
- 限定总资金：用户输入的是包含买入费用在内的现金上限。每格现金额度为 `总资金 ÷ 网格档数`，每格证券本金预算为 `每格现金额度 ÷ (1 + 单边费率)`。

数量取整后，实际投入通常低于计划现金上限，差额作为未使用现金显示，不重新分配到其他档位。固定每格金额模式下，页面同时显示证券本金预算合计和包含实际买入费用的现金需求；限定总资金模式下，实际现金占用不得超过用户输入的总资金。

### 4. 数量与利润

每档统一按 100 份向下取整：

```text
买卖数量 = floor(每格证券本金预算 ÷ 买入价 ÷ 100) × 100
实际买入金额 = 买卖数量 × 买入价
卖出金额 = 买卖数量 × 卖出价
买入费用 = 实际买入金额 × 单边费率
卖出费用 = 卖出金额 × 单边费率
税费前利润 = 卖出金额 - 实际买入金额
税费后利润 = 税费前利润 - 买入费用 - 卖出费用
净收益率 = 税费后利润 ÷ 实际买入金额
```

网格表显示：档位、买入价、卖出价、计划预算、买卖数量、实际买入金额、卖出金额、税费前利润、税费后利润、净收益率。

### 5. 跳空说明

工具只生成计划，不模拟成交。结果区固定说明：若价格跳空跨越多档，应逐档执行；每一档仍按自己的卖出价退出。例如直接从 1.00 跌到 0.90，应执行 1.00、0.95、0.90 三档，反弹后分别按 0.95、1.00、1.05 的对应关系退出。

## 压力测试

压力测试假设所有买入档均已触发，且当前价格停在最低买入价。摘要显示：

- 实际理论覆盖跌幅；
- 网格档数和最低买入价；
- 证券本金预算与包含买入费用的现金需求；
- 最大实际投入与未使用现金；
- 全部触发后的总持仓数量；
- 含买入费用的平均持仓成本；
- 最低档价格下的持仓市值；
- 最低档价格下的浮动盈亏和相对投入成本的浮亏比例。

压力测试公式：

```text
最大实际投入 = sum(各档实际买入金额 + 各档买入费用)
总持仓 = sum(各档买卖数量)
平均成本 = 最大实际投入 ÷ 总持仓
最低档市值 = 总持仓 × 最低买入价
浮动盈亏 = 最低档市值 - 最大实际投入
浮亏比例 = 浮动盈亏 ÷ 最大实际投入
```

## 校验与错误处理

- 起始价、步长比例、最大跌幅和当前资金模式对应的金额必须大于 0。
- 步长比例和最大跌幅必须小于 100%。
- 单边费率不得小于 0。
- 向下覆盖后的最低原始价格必须大于 0。
- 三位小数取整后的买入价必须严格递减；出现重复价格时提示步长过小。
- 每一档至少可以买入 100 份；任一档数量为 0 时不生成计划，并提示提高资金或减少网格档数。
- 输入错误显示在参数区附近，不保留部分生成结果。
- 计算结果使用有限精度数值；金额展示保留两位小数，比例展示保留两位小数。

## 视觉与可用性

- 桌面端参数区可使用两列，窄屏自动变为单列。
- 资金模式切换时只显示当前有效的金额输入，避免两个金额互相冲突。
- 压力测试置于网格表之前，并用清晰的损失色展示最坏情形浮亏。
- 网格表在窄屏允许横向滚动，不压缩到难以阅读。
- 不使用“推荐买入”“安全收益”等暗示性措辞；页面明确说明结果是计划计算，不构成投资建议。

## 测试与验收

### 文章示例

输入：起始价 1.000、步长 5%、最大跌幅 30%、固定每格 10,000 元、费率 0%。

必须生成 7 档买入价：`1.000、0.950、0.900、0.850、0.800、0.750、0.700`；卖出价依次为 `1.050、1.000、0.950、0.900、0.850、0.800、0.750`。

由于首版固定按 100 份取整，数量会与文章图片中按 1 份取整的示例不同，这是已确认的产品差异；价格关系和固定每格预算逻辑必须一致。

### 自动测试

至少覆盖：

1. 文章示例价格序列；
2. 最大跌幅不是步长整数倍时向外覆盖；
3. 固定每格金额与限定总资金两种模式；
4. 100 份向下取整；
5. 双边费用和净收益率；
6. 压力测试总持仓、平均成本和最低档浮亏；
7. 资金不足 100 份；
8. 三位小数取整后出现重复网格；
9. 非法负数、零值以及最低价格不大于零。

### 浏览器验收

- 本地双击 `index.html` 后无需网络即可使用；
- 默认值与资金模式切换正确；
- 输入错误有明确提示，修正后可重新生成；
- 压力摘要和表格与自动测试结果一致；
- 桌面宽屏与窄屏均可操作；
- 浏览器控制台无错误。
